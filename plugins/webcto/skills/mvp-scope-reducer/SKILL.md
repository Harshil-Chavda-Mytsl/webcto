# MVP Scope Reducer

Cut broad platform dreams into something buildable and testable.

## Workflow

1. Identify the core user and urgent job.
2. Keep only features needed to complete that job once.
3. Move enterprise, scale, billing, automation, and polish to later phases unless required.
4. Define the cheapest proof that users care.
5. Turn the MVP into a build checklist.

## Output

Return:

- keep now
- defer
- remove
- assumptions
- MVP acceptance criteria
- first implementation tasks

Treat repeated generic requirements as one quality standard, not dozens of features.
