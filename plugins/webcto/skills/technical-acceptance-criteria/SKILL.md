# Technical Acceptance Criteria

Convert loose requirements into testable finish lines.

## Template

For each feature define:

- user outcome
- inputs and outputs
- happy path
- edge cases
- failure behavior
- security/privacy requirement
- telemetry/logging
- tests
- definition of done

## Rules

- Prefer observable behavior over implementation wishes.
- Include negative cases and permission boundaries.
- Add performance or scale criteria only when measurable.
- Avoid "production grade" without specific checks.
